# Автоматическая обработка корпоративной почты

Проект читает письма из папки, переводит их в JSON, классифицирует по правилам и раскладывает исходные файлы по папкам категорий.

## Категории

* `incidents`: аварии, сбои сервисов, ошибки 500, недоступность систем, остановка работы
* `service_requests`: запросы доступа, учетные записи, рабочее место, заявки на подключение сервисов
* `spam`: фишинг, подозрительные ссылки, просьбы ввести пароль или данные карты, розыгрыши и призы
* `hardware_faults`: принтеры, ноутбуки, мониторы, гарнитуры, сканеры и другое оборудование
* `прочее`: письма, где нет уверенного совпадения с основной категорией

## Как работает пайплайн

1. `app/app.py` берет папку с письмами в формате `txt`.
2. `MailParser` выделяет тему, отправителя, получателя, дату, текст, ссылки и вложения.
3. Результат сохраняется в `all_mails.json`.
4. `classifier/main.py` загружает `all_mails.json` и таблицу весов `classifier/config.csv`.
5. Для темы и текста ищутся маркеры из таблицы.
6. Совпадения в теме получают больший вес, потому что тема обычно короче и важнее.
7. Побеждает категория с достаточным баллом и достаточным отрывом от второго места.
8. Если балл слишком маленький или отрыв неубедительный, письмо попадает в `прочее`.
9. Исходные файлы перемещаются в папки категорий, а полный результат сохраняется в `result.json`.

## Быстрый запуск

```bash
./run_classifier.sh mails sorted
```

После запуска в папке `sorted` появятся:

* `all_mails.json`
* `result.json`
* папки категорий с исходными письмами

## Запуск по шагам

```bash
python app/app.py mails sorted/all_mails.json
python classifier/main.py sorted/all_mails.json mails sorted sorted/result.json
```

## Нестандартные ситуации

* Письмо без совпадений попадает в `прочее`.
* Письмо со слишком близкими баллами по нескольким категориям попадает в `прочее`.
* Файл без корректных заголовков все равно получает текст и может быть классифицирован.
* Нечитаемые файлы не останавливают обработку остальных писем.
* Ошибки парсинга сохраняются рядом с JSON в `parse_errors.json`.
* Если в папке категории уже есть файл с таким именем, новое имя получает номер.
